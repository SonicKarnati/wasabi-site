# Build Readiness

## Why This Doc Exists

This document answers a simple question: what can be built now without breaking the product rules? It is the boundary between safe implementation and future work.

## What Is Ready

- Static site structure.
- Header, footer, and navigation.
- Five route templates.
- Placeholder-safe program and impact patterns.
- Read-only contact page structure.
- Accessibility-first typography, spacing, and color tokens.

## What Is Not Ready Yet

- Final contact routing.
- Final program statuses.
- Verified metrics.
- Approved partner and story assets.
- Rights-cleared imagery beyond the logo.
- Privacy and consent language for submission flows.

## What Can Be Built Safely

- Shared page shell.
- Proof-status UI.
- Basic CTA patterns.
- Placeholder content rendering.
- Review-safe copy structures.

## What Should Wait

- Live form submission.
- CMS wiring.
- Extra pages or microsites.
- Final public launch content that still lacks verification.

## Build Rule

If a feature would require a new claim, a new permission, or a new source of truth, it should wait until that source is available.
