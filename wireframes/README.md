# Wireframes README

## What WireMD Is Being Used For

These files are low-fidelity structural wireframes for Phase 2 website planning. They define page purpose, content hierarchy, trust flow, CTA placement, and review questions before any website implementation begins.

## How to Open in Obsidian

Open this project folder as an Obsidian vault or use the existing workspace. The wireframes are plain Markdown files in `wireframes/`, so they are readable without any plugin.

## How to Preview With WireMD

If the WireMD Obsidian plugin is installed, open any wireframe file and use the plugin's preview/render command for fenced `wiremd` blocks. If WireMD is not installed, review the Markdown and fenced blocks directly.

## How to Review

Review:

- Page purpose.
- Content hierarchy.
- Visitor questions answered.
- CTA placement.
- Trust-building flow.
- Whether unsupported claims are marked as placeholders.
- Whether the wireframe supports the intended audience.

## What Not to Judge Yet

Do not review final colors, final typography, animation, production layout, exact copy polish, imagery, spacing, components, or implementation details. These are not final visual designs.

## Important Rule

Every metric, partner reference, testimonial, photo/story, public-sector claim, and third-party quote should remain a placeholder until verified.
