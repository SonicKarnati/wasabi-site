# Wireframe Review Notes

## What to Review

- Page purpose and audience fit.
- Section order and hierarchy.
- Whether every section has a clear reason to exist.
- CTA placement and clarity.
- Trust flow for government, institutional, school, parent, and international visitors.
- Whether placeholders are used for unverified claims.

## What Not to Review Yet

- Final color.
- Final typography.
- Final component design.
- Animations.
- Exact production copy.
- Photography choices.
- Responsive implementation.
- Next.js, React, Tailwind, shadcn, or build decisions.

## Page-by-Page Questions

### Home

- Does the opening explain Wasabi quickly?
- Does it establish credibility without unsupported claims?
- Does it show enough program breadth without feeling scattered?
- Is the next action obvious?

### About

- Does the belief system feel clear and differentiated?
- Does the Bharat-native context feel grounded?
- Does "The Wasabi Moment" make sense?
- What team/founder/advisor information is required?

### Programs / Work

- Are the program categories correct?
- Which programs should be public in version 1?
- Which programs are active, completed, proposed, or campaign concepts?
- What program details are missing?

### Impact

- Which metrics are verified?
- Which projected metrics should remain internal?
- Which case studies can be approved for public use?
- What proof would government/institutional visitors need?

### Contact

- What is the official public email/domain/phone?
- Which inquiry types should be available?
- Should parents/students have a separate enrollment route?
- Should media/sponsors be included now or later?

## Open Content Questions

- Official primary domain and public email.
- Verified metrics.
- Program status for each named initiative.
- Partner/logo permissions.
- Student story and photo consent.
- Advisor/team bios and permissions.
- Public-sector naming permissions.
- Privacy, safeguarding, consent, and accessibility language.
